代理模式

发表于 2020-11-09 分类于 java ， 设计模式 阅读次数：
本文字数： 5.5k 阅读时长 ≈ 5 分钟
代理模式，静态代理，动态代理

代理模式，就是给一个对象提供一种代理对象以控制该对象的访问。在这个模式中，通过创建代理对象作为“替身”替代了原有对象，完成一些本该由原有对象做事情，从而达到控制对象访问的目的。

通俗来讲：代理 = 代替处理，是由另一个对象来代替原对象来处理某些逻辑

例子：房产中介、代跑腿业务、送外卖……

在一些情况下，一个对象的某些方法想要进行评比或者某种逻辑控制，则我们可以通过代理的方式进行。再此能力上，引申出来的作用，也是目前在开发中经常使用的一个作用，就是：“在不修改原对象代码的基础上，对原对象功能进行修改或者增强”，比如AOP。

分类：

  • 静态代理
  • 基于JDK（接口）的动态代理
  • 基于CGLIB（父类）的动态代理

相关概念：

  • 目标类：原对象，需要代理对象控制它的访问，扩展其功能
  • 代理类：代理模式产生的对象，是原对象的替身，已经在原有基础上修改了逻辑，对目标类进行功能增强

  1. 代理模式在Java开发中广泛应用，特别是在框架中底层原理经常涉及到代理模式（尤其是动态代理）
  2. 静态代理和动态代理，实际使用还是动态代理较多，原因是静态代理需要自行手写代码、维护、修改十分麻烦，会额外引入很多工作量，也不能很好的使用配置完成逻辑的指定
  3. 基于JDK和基于cglib，实际使用都会用到。
    • Spring中两种都支持。指定的目标类实现了接口，spring就会自动选择jdk的动态代理。而如果目标类没有实现接口，会使用cglib
    • 开发中，基于JDK使用要求较多，cglib更通用。
    • 如果使用dubbo+zookeeper，底层进行代理时，最好配置cglib代理。因为dubbo会使用基于包名的扫描方式进行类的处理，而JDK的代理类生成的包名类似于com.sun.proxy…格式。实际上需要让代理类和目标类保持同样的包名，所以cglib能保持原包名不变生成代理类

1 静态代理

使用代理对象替代原来的对象，完成原来对象的功能，并实现功能增强

示例：点外卖

顾客下单，外卖小哥接收订单、取餐、送餐。

顾客是目标对象，被代理。

外卖小哥是代理对象，代理顾客进行取餐，以及将餐送到顾客手中。

  • 点外卖接口
// 点餐接口
public interface IOrder {
  public String order(String foodName);
}
  • 目标对象（被代理的对象）
public class Customer implements IOrder{
  @Override
  public String order(String foodName) {
    return "已经下单" + foodName;
  }
}
  • 代理对象
// 代理对象，外卖小哥
public class DeliveryMan implements IOrder{
  // 目标类对象
  private IOrder customer;
  public DeliveryMan2(IOrder customer) {
    this.customer = customer;
  }
  @Override
  public String order(String foodName) {
    String order = customer.order(foodName);
    // 目标对象只有一个点餐功能，通过外卖小哥代理，对目标对象的功能进行了增强丰富
    System.out.println("已经接收到订单，正前往取餐途中……");
    System.out.println("正在派送");
    return  order + "已经送达";
  }
}
  • 测试
@Test
public void demo() {
  // 创建一个顾客对象
  Customer customer = new Customer();
  // 创建代理对象,代理这个顾客
  IOrder deliveryMan = new DeliveryMan2(customer);
  String result = deliveryMan.order("千叶豆腐");
  System.out.println(result);
}

缺点：

静态代理虽然能够实现代理模式，完成解耦，但是静态代理类的代码维护依然非常复杂，一旦接口或者父类发生变动，则代理类的代码就得随之修改，代理类多的时候维护困难，所以实际上动态代理用的更多。

2 动态代理

动态代理技术，是在内存中生成代理对象的一种技术，也就是整个代理过程在内存中进行，不需要手写代理类的代码，也不会存在代理类编写的过程，而是直接在运行期，在JVM中“凭空”创造出的代理类对象供使用，一般使用的动态代理技术是以下两种：

2.1 基于JDK（接口）的动态代理

JDK自带的动态代理技术，需要使用一个静态方法来创建代理对象。它要求被代理对象，也就是目标类，必须实现接口。生成的代理对象和原对象实现的是相同的接口，是兄弟关系

  • IOrder接口
// 点餐接口
public interface IOrder {
  public String order(String foodName);
  public void test1();
  public void test2();
}
  • 目标类
public class Customer implements IOrder {
  @Override
  public String order(String foodName) {
    return "已经下单" + foodName;
  }
  @Override
  public void test1() {}
  @Override
  public void test2() {}
}
  • 测试
@Test
public void test() {
  // 准备一个目标对象
  Customer customer = new Customer();
  // 使用jdk的API，动态生成一个代理对象
  IOrder deliveryMan = (IOrder)Proxy.newProxyInstance(
    customer.getClass().getClassLoader(),
    customer.getClass().getInterfaces(),
    (proxy, method, args)->{
      // 调用目标对象的对应方法
      Object result = method.invoke(customer, args);
      System.out.println(result);
      // 进行代理类的一些增强逻辑
      System.out.println("已经接收到订单，正前往取餐途中……");
      System.out.println("正在派送");
      return args[0] + "已经送达";
    }
  );
  // 调用代理对象,执行对应方法
  String result = deliveryMan.order("麻婆豆腐");
  System.out.println("result:" + result);
}
public static Object newProxyInstance(ClassLoader loader,Class<?>[] interfaces,InvocationHandler h)

ClassLoader loader：固定写法，指定目标类对象的类加载器，用于加载目标类及其接口的字节码文件，通过**对象.getClass().getClassLoader()**获得

Class<?>[] interfaces： 固定写法，指定目标类的实现的所有接口的字节码对象的数组，通过**对象.getClass().getInterfaces()**获得
InvocationHandler h：InvocationHandler 是个接口，其中只有一个invoke方法。它会在代理类对象调用方法时执行，也就是说，在代理类对象中调用接口中任何方法时，都会执行到invoke中。所以，在此方法中完成对增强或扩展代码逻辑的编写。

public interface InvocationHandler {
  /**
   * proxy: 代理类对象的引用，也就是Proxy.newProxyInstance的返回值
   * method: 对应的是触发invoke执行的方法的Method对象，比如上面调用了接口的order方法，这个method就是order的反射对象
   * args: method中传递的参数
   */
  public Object invoke(Object proxy, Method method, Object[] args)throws Throwable;
}

知道了Method、args，那么就可以进行增强操作了。

将上面的动态代理转化为静态代理，相当于如下代码：

// 省略接口定义和Customer类，详情见上面
// 代理对象，外卖小哥，这是在运行中动态创建的代理类，因为在创建时，传递的参数中有给到目标类的实现接口，如果创建的代理类也能够实现同样的接口
public class DeliveryMan implements IOrder{
  // 目标类对象
  private IOrder customer;
  public DeliveryMan(IOrder customer) {
    this.customer = customer;
  }
  @Override
  public String order(String foodName) {
    // 相当于method.invoke(customer, args)，得到目标类的该方法返回结果
    String order = customer.order(foodName);
    // 目标对象只有一个点餐功能，通过外卖小哥代理，对目标对象的功能进行了增强丰富
    System.out.println("已经接收到订单，正前往取餐途中……");
    System.out.println("正在派送");
    // 返回代理对象的order方法结果
    return  order + "已经送达";
  }
}

2.2 基于CGLIB（父类）的动态代理

在内存中，生成一个代理对象，它继承了目标对象。

  • 导包：cglib和asm
<!-- https://mvnrepository.com/artifact/cglib/cglib -->
<dependency>
  <groupId>cglib</groupId>
  <artifactId>cglib</artifactId>
  <version>3.3.0</version>
</dependency>
<!-- https://mvnrepository.com/artifact/org.ow2.asm/asm -->
<dependency>
  <groupId>org.ow2.asm</groupId>
  <artifactId>asm</artifactId>
  <version>9.0</version>
</dependency>
public static Object create(Class type, Callback callback)

Class type：目标类的class属性

Callback callback：回调，提供一个方法，它会在合适的时候调用。由于Callback接口只是作为名称定义的作用，不包含方法声明，所以在使用的时候，大多使用其子接口MethodInterceptor

public interface MethodInterceptor extends Callback
  public Object intercept(Object obj, java.lang.reflect.Method method, Object[] args,MethodProxy proxy) throws Throwable;
}
@Test
public void test() {
  // 准备一个目标对象
  Customer customer = new Customer();
  // 使用cglib创建代理类对象
  Customer deliveryMan = (Customer)Enhancer.create(customer.getClass(), new MethodInterceptor() {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
      if("order".equals(method.getName())) {
        // 调用目标对象的对应方法
        Object result = method.invoke(customer, args);
        System.out.println(result);
        // 进行代理类的一些增强逻辑
        System.out.println("已经接收到订单，正前往取餐途中……");
        System.out.println("正在派送");
        return result + "已经送达";
      } else {
        // 反射调用原来的逻辑
        return method.invoke(customer, args);
      }
    }
  });
  // 调用代理对象,执行对应方法
  String result = deliveryMan.order("麻婆豆腐");
  System.out.println("result:" + result);
}
