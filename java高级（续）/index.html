5.1 UUID类

UUID 是 通用唯一识别码（Universally Unique Identifier）的缩写，是一种软件建构的标准，亦为分布式计算环境领域的一部分。其目的，是让分布式系统中的所有元素，都能有唯一的辨识信息，而不需要通过中央控制端来做辨识信息的指定。如此一来，每个人都可以创建不与其它人冲突的UUID。在这样的情况下，就不需考虑数据库创建时的名称重复问题。目前最广泛应用的UUID，是微软公司的全局唯一标识符（GUID

UUID由以下几部分的组合：

（1）当前日期和时间，UUID的第一个部分与时间有关，如果你在生成一个UUID之后，过几秒又生成一个UUID，则第一个部分不同，其余相同。

（2）时钟序列。

（3）全局唯一的IEEE机器识别号，如果有网卡，从网卡MAC地址获得，没有网卡以其他方式获得。

（4）在 hibernate（Java orm框架）中， 采用 IP-JVM启动时间-当前时间右移32位-当前时间-内部计数（8-8-4-8-4）来组成UUID

public static UUID randomUUID();// 获取UUID对象
public static UUID fromString(String name);// 根据字符串获取UUID内容

5.2 Optional类

在接受对象的一方，往往需要做判空处理，为了解决这个情况，在JDK1.8提供了Optional类，可以实现null的处理。

// 返回空数据
public static<T> Optional<T> empty(){};
// 获取数据
public T get(){};
// 保存数据,但是不允许出现null,出现null会出现空指针异常
public static <T> Optional<T> of(T value);
// 保存数据允许为空
public static <T> Optional<T> ofNullable(T value);
// 空的时候返回其他数据
public T orElse(T other){};

class MessageUtil {
	private MessageUtil() {}

	public static Optional<IMessage> getMessage() {
    // 倘若Optional中保存null需要用ofNullable,如果使用of会出现空指针异常
		return Optional.ofNullable(null);
	}
	public static void useMessage(IMessage msg) {
		if(msg != null)
			System.out.println(msg.getContent());
	}
}

interface IMessage {
	public String getContent();
}

class MessageImpl implements IMessage {
	@Override
	public String getContent() {
		return "hahhah";
	}
}

public class MyTest {
	public static void main(String[] args) {
		// 获取数据,倘若数据为null,在get的时候会出现java.util.NoSuchElementException
		IMessage message = MessageUtil.getMessage().get();
    // 改进上面的写法,使用orElse,倘若Optional中保存为null,则创建
    message = MessageUtil.getMessage().orElse(new MessageImpl());
		MessageUtil.useMessage(message);
	}
}

5.3 ThreadLocal类

在了解之前，先看下面的例子

// 要发送的消息
class Message {
	private String info;

	public String getInfo() {
		return info;
	}

	public void setInfo(String info) {
		this.info = info;
	}
}
// 消息的发送通道
class Channel {
	private static Message message;

	public static void send() {
		System.out.println("【"+Thread.currentThread().getName()+"消息发送：】" + message.getInfo());
	}

	public static void setMessage(Message message) {
		Channel.message = message;
	}
}
public class MyTest {
	public static void main(String[] args) {
		Message msg = new Message(); // 实例化消息主体对象
		msg.setInfo("haha"); // 设置发送内容
		Channel.setMessage(msg); // 设置要发送的消息
		Channel.send(); // 发送消息
	}
}

对于上面的例子，采用的是单线程模式来进行处理的。那么在多线程的情况下，能否实现同样的效果？为此，启动30个线程进行处理

for(int i = 0; i < 30; i++) {
  int x = i;
  new Thread(()->{
    Message msg = new Message(); // 实例化消息主体对象
    msg.setInfo(x +"haha"); // 设置发送内容
    Channel.setMessage(msg); // 设置要发送的消息
    Channel.send(); // 发送消息
  }, "消息发送者"+ i).start();
}

测试后发现，线程发送的消息出现混乱。因为每个线程都会对静态对象message设置其中的info内容，当其中一个线程修改其内容还没有进行发送，另一个线程就将内容修改，所以会出现数据覆盖，不同步的问题。

在Channel核心结构不改变的情况下，要考虑每个线程独立操作问题，发现对于Channel类而言除了保留发送消息之外，还应该多存放一个每个线程的标记，JDK1.2后可以通过ThreadLocal类来进行存放。

// 构造方法
public ThreadLocal() {}
// 取出数据
public T get();
// 设置数据
public void set(T value);
// 移除数据，@since:1.5
public void remove() 

class Channel {
  private static final ThreadLocal<Message> THREAD_LOCAL = new ThreadLocal<Message>();

  public static void send() {
    // 通过get获取ThreadLocal存放的对象,此获取是通过当前线程currentThread做key来获取的
    System.out.println("【"+Thread.currentThread().getName()+"消息发送：】" + THREAD_LOCAL.get().getInfo());
  }

  public static void setMessage(Message message) {
    // 向THREAD_LOCAL中保存数据,同时也会保存Thread.currentThread
    THREAD_LOCAL.set(message); 
  }
}

ThreadLocal.get获取当前线程存储的对象，是将当前线程做key，对象做value，因为ThreadLocal类有内部类ThreadLocalMap，用于存放key-value

5.4 定时调度

定时器主要操作是进行定时任务的处理，不过在java中提供定时任务支持，不过只是实现了一种间隔触发的操作。

如果想要实现定时操作的主体类，以及一个定时任务的控制，JDK1.3后可以使用两个类实现：

  • TimerTask抽象类：实现Runnable接口，实现定时任务处理
  • Timer类：进行任务的启动
    • 启动方法：public void schedule(TimerTask task, long delay)，延迟单位毫秒
    • 间隔触发：public void schedule(TimerTask task, long delay, long period)

class MyTask extends TimerTask {
  @Override
  public void run() {
    System.out.println(Thread.currentThread().getName() + "定时任务当前时间"+ System.currentTimeMillis());
  }
}
public class MyTest {
  public static void main(String[] args) {
    Timer timer = new Timer();
    // timer.schedule(new MyTask(), 0);
    timer.schedule(new MyTask(), 100, 1000);
  }
}

5.5 Base64加密

倘若需要解密，需要知道加密的规则，在JDK1.8开始，提供一种新的加密处理操作类，即Base64类，里面有两个内部类，Encoder和Decoder。

String msg = "haha,silverbeats";
// 得到加密后的byte数组
byte[] encode = Base64.getEncoder().encode(msg.getBytes());
// 将byte数组转为String
String encMsg = new String(encode);
// 打印加密结果
System.out.println(encMsg);
// 将加密的encMsg解密
byte[] decode = Base64.getDecoder().decode(encMsg);
System.out.println(new String(decode));

因为Base64是公版加密算法，直接对数据加密不安全，可以对原始数据加上盐值，如：

String salt = "learn";
String msg = "haha,silverbeats" + "{"+ salt +"}";

最好的方法是多次加密

class StringUtil {
  // 公共盐值
  private static final String SALT = "learn";
  // 加密次数
  private static final int REPEAT_TIMES = 5;
  /**
	 * @param pwd:需要加密的数据
	 * @return: 返回加密后的String字符串
	 * @description: 对数据进行加密
	 */
  public static String enCode(String pwd) {
    // 拼接盐值
    String temp = pwd + "{" + SALT + "}";
    // 转字节数组
    byte[] tempBytes = temp.getBytes();
    // 多次加密
    for (int i = 0; i < REPEAT_TIMES; i++)
      tempBytes = Base64.getEncoder().encode(tempBytes);
    // 返回结果
    return new String(tempBytes);
  }

  public static String deCode(String code) {
    // 获取加密后的字节数组
    byte[] codeBytes = code.getBytes();
    // 多次解密
    for (int i = 0; i < REPEAT_TIMES; i++)
      codeBytes = Base64.getDecoder().decode(codeBytes);
    // 转字符串
    String tempRes = new String(codeBytes);
    // 去掉尾部盐值
    return tempRes.substring(0, tempRes.indexOf('{'));
  }
}

5.6 Comparable比较器

所谓比较器指的就是进行大小关系的判断。

class Person {
	private String name;
	private int age;

	public Person(String name, int age) {
		this.name = name;
		this.age = age;
	}
}
public class MyTest {
	public static void main(String[] args) {
		Person[] data = new Person[]{
      new Person("name1",40), 
      new Person("name2", 30)
    };
		Arrays.sort(data);
		System.out.println(Arrays.toString(data));
	}
}

上面代码运行报错：java.lang.ClassCastException: Thread.Person cannot be cast to java.lang.Comparable

任意一个类默认情况下无法进行数组排序或比较需求的，可以实现Comparable接口，自定义实现比较规则。

class Person implements Comparable<Person> {
	@Override
	public int compareTo(Person o) {
    // 当前数据比传入的对象小：返回负数
    // 当前数据比传入的对象大：返回正数
    // 当前数据比传入的对象等：返回0
		return this.age - o.age;
	}
}

5.7 Comparator比较器

Comparator属于一种挽救的比较器支持，其主要目的是解决一些没有使用Comparable排序的类的对象数组排序。

比如上面的Person类，没有想过排序，但是过了几天后，需要对Person进行排序，但是不能修改Person类，就是说不能重写compareTo方法。在Arrays类里面排序有另外一种基于Comparator实现方式。

public static <T> void sort(T[] a, int fromIndex, int toIndex, Comparator<? super T> c)
class PersonComparator implements Comparator<Person> {
	@Override
	public int compare(Person o1, Person o2) {
		return o1.getAge() - o2.getAge();
	}
}
Arrays.sort(data, new PersonComparator());

7.3 字节流和字符流区别

  • 在使用OutputStream和Writer输出的最后发现都使用了close方法进行关闭流。

  • 当使用OutputStream进行输出，未close关闭，内容仍然能输出成功；然而使用Writer输出时，没有使用close方法进行关闭，内容无法进行输出，因为Writer使用了缓冲区，当使用close后，会强制进行缓冲的刷新，如果再不关闭的情况下，仍然输出成功，可以使用flush方法。

  • 使用缓冲区的字符流更加适合进行中文数据的处理

7.6 内存流

之前的都是文件操作流，以文件为中心，进行读和写。内存流是指以内存为中心，进行读和写。Java提供了字节内存操作流和字符内存操作流。

  • 字节内存操作流：ByteArrayInputStream，ByteArrayOutputStream
  • 字符内存操作流：CharArrayReader、CharArrayWriter

ByteArrayInputStream构造：public ByteArrayInputStream(byte buf[])

ByteArrayOutputStream构造：public ByteArrayOutputStream()

ByteArrayOutputStream可以通过构造指针缓冲区的大小，默认是32

ByteArrayOutputStream还有其独有的方法，可以一次性得到将缓冲区所有内容的方法：

public synchronized byte toByteArray()[] {
    return Arrays.copyOf(buf, count);
}
public synchronized String toString() {
  return new String(buf, 0, count);
}
public static void main(String[] args) throws Exception{
  // 需要先设置输入信息
  String str = "hello";
  // 创建输入内存流
  try(
    InputStream input = new ByteArrayInputStream(str.getBytes("utf-8"));
    ByteArrayOutputStream out = new ByteArrayOutputStream()
  ) {
    int data = 0;
    while((data = input.read()) != -1) {
      // 向输出流保存数据
      out.write(Character.toUpperCase((char)data));
    }
    // 一次性获得内存输出流中缓冲区所有内容
    byte[] bytes = out.toByteArray();
    System.out.println(new String(bytes, "utf-8"));
    System.out.println(out);
  } catch (IOException e) {
    e.printStackTrace();
  }
}

7.7 管道流

实现两个线程之间的IO处理。

  • 字节管道流：PipedInputStream、PipedOutputStream
  • 字符管道流：PipedReader、PipedWriter
class SendThread implements Runnable {
	// 管道输出流
	private PipedOutputStream output;

	public SendThread() {
		this.output = new PipedOutputStream();
	}

	public PipedOutputStream getOutput() {
		return output;
	}

	@Override
	public void run() {
		for (int i = 0; i < 10; i++) {
			String data = "【第" + (i + 1) + "次信息发送】" + Thread.currentThread().getName() + '\n';
			try {
				// 利用管道实现数据发送
				this.output.write(data.getBytes("UTF-8"));
			} catch (IOException e) {
				e.printStackTrace();
			}
		}
		try {
			this.output.close();
		} catch (IOException e) {
			e.printStackTrace();
		}
	}
}

class ReceiveThead implements Runnable {
	// 管道输入流
	private PipedInputStream input;

	public ReceiveThead() {
		this.input = new PipedInputStream();
	}

	public PipedInputStream getInput() {
		return input;
	}

	@Override
	public void run() {
		byte[] data = new byte[1024];
		ByteArrayOutputStream bos = new ByteArrayOutputStream();
		int len;
		try {
			while ((len = this.input.read(data)) != -1) {
				bos.write(data, 0, len);
			}
			System.out.println("【接受消息】" + new String(bos.toByteArray(), "UTF-8"));
			this.input.close();
			bos.close();
		} catch (IOException e) {
			e.printStackTrace();
		}

7.8 随机流RandomAccessFile

倘若一个十分庞大的文件需要读取，传统IO操作不能完成，java.io中提供RandomAccessFile类，可以实现文件跳跃式的读取，比如：可以读取中间的部分内容（前提是：需要一个完善的保存形式）

public RandomAccessFile(File file, String mode)
  • mode：文件处理模式，r，rw，rwd，rws
public static void main(String[] args) throws Exception {
  File file = new File("E:/test/hello.txt");
  RandomAccessFile raf = new RandomAccessFile(file, "rw");
  String[] names = new String[]{"zhangsan", "wangwu  ", "lisi    "};
  int[] age = new int[]{30, 20, 16};
  for(int i = 0; i < names.length; i++) {
    raf.write(names[i].getBytes("utf-8"));
    raf.writeInt(age[i]);
  }
  raf.close();
}

该类最大的特点是在读取，因为数据都是按照固定要求存储的，所以可以实现跳跃读取。可以看到上面的例子中往文件中写的内容长度固定是12个字符，字符数据占8个，整型int占4位。所以，可以使用随机流，跳过部分内容，直接读取最后的“lisi 16”

public static void main(String[] args) throws Exception {
  File file = new File("E:/test/hello.txt");
  RandomAccessFile raf = new RandomAccessFile(file, "rw");
  // 李四前面有张三和王五的两行数据，长度24，需要跳过，指定开始读取的位置
  raf.skipBytes(24);
  // 读取的是lisi    名字
  byte[] data = new byte[8];
  int len = raf.read(data);
  System.out.println("姓名：" + new String(data, 0, len));
  // 继续读取后面4位的年龄
  System.out.println("年龄：" + raf.readInt());
  // 上面已经读到李四的年龄16了,倘若继续想要读取张三的数据,那么需要让读取位置往回走
  // 重新指定读取位置
  raf.seek(0);
  len = raf.read(data);
  System.out.println("姓名：" + new String(data, 0, len).trim());
  System.out.println("年龄：" + raf.readInt());
}

HashMap源码：

static final float DEFAULT_LOAD_FACTOR = 0.75f;
public HashMap() {
  // 充填因子默认0.75
  this.loadFactor = DEFAULT_LOAD_FACTOR; 
}
public V put(K key, V value) {
  // 对key进行hash处理,生成hash码
  // putVal方法中，提供一个Node节点进行数据的保存
  // 而在使用putVal方法过程中，会调用resize的方法，可以进行容量的扩充
  return putVal(hash(key), key, value, false, true);
}

HashMap的put操作如何实现的扩充：

  • 在HashMap类中，提供一个初始容量大小的常量，值为16，也就是说，默认可以保存最大内容是16
  • 当保存的内容数量超过阈值（0.75）时，会进行扩充，调用resize方法
  • 在扩充的时候，进行成倍扩充

HashMap工作原理：

在HashMap之中进行数据存储是利用Node类完成，这种情况下，说明这种情况只能是链表或者二叉树，在数据量不大的情况下，链表查找的时间复杂O(n)，与树的查找时间复杂度O(logn)差别不是特别大。

但是在JDK1.8后，HashMap的实现出现改变，需要适应大数据，其存储发生变化。在HashMap之中有一个常量：static final int TREEIFY_THRESHOLD = 8，在保存的数据个数没有超过阈值8时，会按照链表存储，一旦超过，会将链表保存为红黑树，利用左旋与右旋实现平衡，提高查找效率。

8.7 Stream数据流

Stream主要功能是进行数据的分析，同时针对集合中的数据进行分析操作

public interface Collection<E> extends Iterable<E> {
  default Stream<E> stream() {
    return StreamSupport.stream(spliterator(), false);
  }
}
List<String> all = new ArrayList<>();
Collections.addAll(all,"Java", "JavaScript", "Python", "Spring", "SpringMVC", "Mybatis", "SpringBoot");
// 获取Stream接口对象
Stream<String> stream = all.stream();
// 输出元素个数
System.out.println(stream.count());
// 将stream中每个元素都转为小写,并统计其中包含"j"的元素个数
System.out.println(stream.filter(ele -> ele.toLowerCase().contains("j")).count());
// 一种链式调用
List<String> collect = stream
  // 过滤
  .filter(ele -> ele.toLowerCase().contains("s"))
  // 去重复
  .distinct()
  //跳过2个
  .skip(2)
  // 取出2个
  .limit(2)
  // 收集到List集合中
  .collect(Collectors.toList());

8.8 MapReduce模型

在进行数据分析处理之中，有一个MapReduce模型，这个模型分为：Map处理部分和Reduce分析部分，在分析之前，需要对数据进行处理。

@ToString
@Getter
@Setter
@AllArgsConstructor
class Order {
	private String name;
	private double price;
	private int amount;
}
public class MyTest {
  public static void main(String[] args) throws Exception {
    List<Order> all = new ArrayList<Order>();
    all.add(new Order("娃娃1", 9, 10));
    all.add(new Order("娃娃2", 2000, 3));
    all.add(new Order("电脑", 9000, 8));
    all.add(new Order("茶杯", 2.9, 800));
    all.add(new Order("蛋糕", 10, 140));
    // 购买商品中,是娃娃的信息数据,并进行商品单价和数量的处理,随后分析汇总
    DoubleSummaryStatistics summaryStatistics = all.stream()
      .filter(ele -> ele.getName().contains("娃"))
      .mapToDouble(orderObject -> orderObject.getPrice() * orderObject.getAmount())
      .summaryStatistics();
    System.out.println("订单数量："+ summaryStatistics.getCount());
    System.out.println("购买总价："+ summaryStatistics.getSum());
    System.out.println("平均花费："+ summaryStatistics.getAverage());
    System.out.println("最高花费："+ summaryStatistics.getMax());
    System.out.println("最低花费："+ summaryStatistics.getMin());
  }
}

9.1 statement的问题

在statement接口里面如果想要执行SQL语句，需要使用字符串实现SQL，如果要结合用户输入数据的情况下，会存在问题：

  • sql拼接处理麻烦，编写维护困难
  • 不能很好的描述日期形式
  • 对于敏感字符数据无法进行合理拼凑

9.2 PreparedStatement

为了解决Statement接口存在的缺点，java.sql包里面提供PreparedStatement子接口。

public interface PreparedStatement extends Statement 
String sql = "insert into user(id, name, pwd) values(?,?,?)";

PreparedStatement prepareStatement = connection.prepareStatement(sql);
// 在执行数据库操作之前,为占位符设置内容
prepareStatement.setInt(1, 20);
prepareStatement.setString(2, "silverbeats");
prepareStatement.setString(3, "haha");
// prepareStatement.setDate();
// prepareStatement.setDouble();
prepareStatement.executeUpdate();

prepareStatement.close();
  • 查询全部数据
String sql = "select * from user";

PreparedStatement prepareStatement = connection.prepareStatement(sql);

ResultSet resultSet = prepareStatement.executeQuery();
while(resultSet.next()) {
  System.out.println(resultSet.getInt("id") + resultSet.getString("name") + resultSet.getString("pwd"));
}
  • 根据id查数据
String sql = "select * from user where id = ?";

PreparedStatement prepareStatement = connection.prepareStatement(sql);
prepareStatement.setInt(1, 20);
ResultSet resultSet = prepareStatement.executeQuery();

while(resultSet.next()) {
  System.out.println(resultSet.getInt("id")+ resultSet.getString("name") + resultSet.getString("pwd"));
}
  • 分页查找
String sql = "select * from user limit ?,?";

PreparedStatement prepareStatement = connection.prepareStatement(sql);
prepareStatement.setInt(1, (currentPage-1)*pageSize);
prepareStatement.setInt(2, pageSize);

ResultSet resultSet = prepareStatement.executeQuery();
