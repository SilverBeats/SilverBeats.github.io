java题目汇总

1 多线程题目

1.1 Runnable与Callable区别

Runnable与Callable区别

  • Runnable是在JDK1.0是否提出的多线程的接口实现，Callable是在JDK1.5之后提出的
  • Runnable只提供run方法，没有返回值
  • Callable接口提供call方法，可以有返回值

1.2 volatile与synchronized区别

volatile与synchronized区别

  • volatile主要在属性上使用，synchronized用户代码块与方法上使用
  • volatile无法描述同步的处理，只是一种直接内存的处理，避免副本操作
  • synchronized是实现同步的

2 Java类库题目

2.1 String、StringBuffer与StringBuilder区别

String、StringBuffer与StringBuilder区别

  • String类是字符串的首选，其最大特点是内容不可修改
  • StringBuilder和StringBuffer类的内容允许修改
  • StringBuffer是在JDK1.0就有的，StringBuilder是在JDK1.5之后提供
  • StringBuffer线程安全，方法都有synchronized关键字进行标注，StringBuilder线程不安全

2.2 Comparable与Comparator区别

Comparable与Comparator区别

  • java.lang.Comparable是在类定义的时候实现的父接口，主要用于定义排序规则，里面只有compareTo方法
  • java.util.Comparator是挽救比较器操作，需要设置单独的比较器规则类实现排序，里面有compare方法

3 集合题目

3.1 ArrayList、LinkedList、Vector各自的特点

ArrayList、LinkedList、Vector各自的特点

ArrayList

  • 底层使用Object数组实现
  • 使用无参构造创建对象时，使用空数组，在这种情况下再执行add添加，才会创建一个容量为10的Object数组
  • 扩充时，新数组大小是原先的1.5倍。比如原先10，扩充后是15
  • 扩充时底层使用System.arraycopy
  • list集合定义通过index来get某个位置的元素，时间复杂度O(1)

LinkedList

  • 底层是双向链表，每个节点有pre和next指针
  • list集合定义通过index来get某个位置的元素，时间复杂度O(n)

Vector

  • 继承结构与ArrayList一致
  • 每个操作方法都有同步关键字synchronized，线程安全
  • 使用无参构造时，会创建一个容量10的Object数组

3.2 Collection.remove和Iterator.remove区别

Collection.remove和Iterator.remove区别

进行迭代输出时，如果使用了Collection.remove方法会造成并发更新异常，导致程序出错，而使用后者可以实现正常删除

3.3 HashMap的put操作如何实现的扩充

HashMap的put操作如何实现的扩充

  • 在HashMap类中，提供一个初始容量大小的常量，值为16，也就是说，默认可以保存最大内容是16
  • 当保存的内容数量超过阈值（0.75）时，会进行扩充，调用resize方法
  • 在扩充的时候，进行成倍扩充

3.4 HashMap的工作原理

HashMap的工作原理

在HashMap之中进行数据存储是利用Node类完成，这种情况下，说明这种情况只能是链表或者二叉树，在数据量不大的情况下，链表查找的时间复杂O(n)，与树的查找时间复杂度O(logn)差别不是特别大。

但是在JDK1.8后，HashMap的实现出现改变，需要适应大数据，其存储发生变化。在HashMap之中有一个常量：static final int TREEIFY_THRESHOLD = 8，在保存的数据个数没有超过阈值8时，会按照链表存储，一旦超过，会将链表保存为红黑树，利用左旋与右旋实现平衡，提高查找效率。

3.5 HashTable与HashMap之间的区别

HashTable与HashMap之间的区别

  • HashMap线程不安全，效率高，允许key或者value为null

  • HashTable线程安全，效率低，不允许key或者value为null

  • HashTable有线程同步检测synchronized
